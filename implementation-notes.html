同步补丁的结尾换行规范化

状态：agent-resolved

日期：2026-07-12

背景：overrides.yaml 中使用 YAML block scalar（text: |）定义的 append_content 文本自带结尾换行；补丁引擎此前又无条件追加一个换行，导致强制同步审计生成仅多一个尾部空行的差异，并错误触发 plugin 版本升级和 PR。

决策：append_content 在拼接前对补丁文本执行 trimEnd()，最终统一保留一个文件结尾换行。该规范只消除补丁配置携带的尾部空白，不改变正文内部空白。

验证：tests/patch-engine.test.ts 覆盖带结尾换行的 YAML block scalar；同步 Action run 29180216479 重新审计后，原误报的 docs-vs-code、docs-vs-docs、geju、goudi、idea、razor、web 均报告“文件内容无变化”。
