同步补丁的结尾换行规范化

状态：agent-resolved

日期：2026-07-12

背景：overrides.yaml 中使用 YAML block scalar（text: |）定义的 append_content 文本自带结尾换行；补丁引擎此前又无条件追加一个换行，导致强制同步审计生成仅多一个尾部空行的差异，并错误触发 plugin 版本升级和 PR。

决策：append_content 在拼接前对补丁文本执行 trimEnd()，最终统一保留一个文件结尾换行。该规范只消除补丁配置携带的尾部空白，不改变正文内部空白。

验证：tests/patch-engine.test.ts 覆盖带结尾换行的 YAML block scalar；同步 Action run 29180216479 重新审计后，原误报的 docs-vs-code、docs-vs-docs、geju、goudi、idea、razor、web 均报告“文件内容无变化”。

Skill 分发载荷精简规则

状态：user-decided

日期：2026-07-22

背景：上游同步会把开发测试、已渲染 golden、仓库门面文档和未接入主执行路径的参考资料一并带入 plugin；直接删除会在后续同步时恢复。

决策：通过 overrides.yaml 的 exclude_files 永久屏蔽 archify 测试与 rendered HTML、kami 测试/发版/MCP 脚本、无用 README 和 .gitignore、goal-gen manifest、六份不可达的 pua 高阶参考，以及 hai 系列的 SKILL.zh_CN.md。保留各平台 agent metadata、xiaohei 示例图、web worked example、teach glossary 和所有许可证文件。

实现约束：exclude_files 中以 / 结尾的规则按目录前缀匹配，以便持续屏蔽后续新增的同类文件；被排除文件不得继续作为 target_patches 目标，并须从 .sync-state.json 的文件清单移除。布局测试持续验证排除规则、同步状态与发布目录一致。

决策来源：用户明确选择上述保留与删除范围，并要求所有删除通过同步规则屏蔽，避免后续同步重新带回。
